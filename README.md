## 🖥️ What this does

autoeq-mcp is an MCP server for the AutoEQ headphone equalization database. It helps you search for headphones and IEMs, compare results, and get EQ settings for more than 8,800 models.

It is useful when you want:

- EQ settings for a specific headphone or IEM
- a quick way to compare models
- a simple way to look up AutoEQ data from an AI tool that supports MCP
- less time spent digging through database pages

## 🔌 Connecting to an MCP client

To use autoeq-mcp inside another app:

1. Open the app that supports MCP.
2. Go to its settings or tools area.
3. Add a new MCP server.
4. Set the server name to autoeq-mcp.
5. Point the app to the local server address or command shown in the project setup.
6. Save the settings.
7. Test the connection by searching for a headphone or IEM.

Common MCP clients may ask for:

- a command to start the server
- a local address like localhost
- a config file path
- a name for the server entry

## 🛠️ Troubleshooting

If the app does not start:

1. Check that you downloaded the right file.
2. Try running it again after the first failed attempt.
3. Make sure Windows did not block the file.
4. Check that your MCP client points to the right server.
5. Restart the app and try another search.

If the app opens but cannot find data:

1. Check the spelling of the headphone or IEM name.
2. Try a shorter model name.
3. Use a well-known model first to test the connection.
4. Confirm that the server is still running.

If your MCP client cannot connect:

1. Verify the server is enabled.
2. Check the server address or command.
3. Make sure the client supports MCP.
4. Restart both the server and the client.
